Soccer Database:

Los resultados en los partidos de fútbol pueden ser consecuencia de varios factores distintos, sin embargo, existen indicadores numéricos que pueden ayudar a predecir un resultado previo a un encuentro con cierta probabilidad. El presente proyecto contiene una amplia base de datos de partidos de ligas europeas las cuales son analizadas usando SQL.

Database info:

La base de datos contiene información a cerca de distintas ligas de fútbol a lo largo de varias temporadas, organizada en las siguientes tablas:

Country              Player               Team               
League               Player_Attributes    Team_Attributes    
Match                Tabla de posiciones

La tabla 'Country' contiene la información a cerca de los distintos países cuya liga se encuentra en la base de datos. La estructura de la base de datos es la siguiente:

CREATE TABLE `Country` (
	`id`	INTEGER PRIMARY KEY AUTOINCREMENT,
	`name`	TEXT UNIQUE
);

La tabla 'Team' contiene información a cerca de los equipos que tienen al menos un partido en la base de datos. La estructura de la tabla es la siguiente:

CREATE TABLE IF NOT EXISTS "Team" (
	`id`	INTEGER PRIMARY KEY AUTOINCREMENT,
	`team_api_id`	INTEGER UNIQUE,
	`team_fifa_api_id`	INTEGER,
	`team_long_name`	TEXT,
	`team_short_name`	TEXT
);

La tabla 'Player' contiene información a cerca de los jugadores que participaron en los encuentros registrados en la base de datos. Su estructura es:

CREATE TABLE `Player` (
	`id`	INTEGER PRIMARY KEY AUTOINCREMENT,
	`player_api_id`	INTEGER UNIQUE,
	`player_name`	TEXT,
	`player_fifa_api_id`	INTEGER UNIQUE,
	`birthday`	TEXT,
	`height`	INTEGER,
	`weight`	INTEGER
);

La tabla 'League' contiene la información correspondiente a las ligas registradas en la base de datos. Su estructura es:

CREATE TABLE `League` (
	`id`	INTEGER PRIMARY KEY AUTOINCREMENT,
	`country_id`	INTEGER,
	`name`	TEXT UNIQUE,
	FOREIGN KEY(`country_id`) REFERENCES `country`(`id`)
);
